Drafted-unproven: no lab has consumed it yet.
- ◐ **Milestone 2 — first live provider** (2026-07-03; **live path proven 2026-07-19**): OpenAI `gpt-image-1.5` provider (native alpha; **1.5 not `gpt-image-2`** — 2 dropped transparency support; 1.5 chosen over the plan's original `gpt-image-1` because it keeps alpha *and* is newer/better) + Godot pixel-art `.import` sidecars (`godot_import.py`) + `--live` smoke script (`scripts/smoke_live.py`, excluded from gate; needs `--extra live` + `OPENAI_API_KEY`). 48 tests, gate green. **Live API call now exercised** (`smoke_live.py`, real key): one `sprite` ran generate → postprocess → checks → stub judge and **passed all deterministic checks** (32×32 RGBA, 100% transparent border, all pixels on the 6-color palette); the `.import` sidecar wrote correctly. Confirms the muddy-downscale tradeoff — gpt-image at 1024→32 is soft-edged, not pixel-native, which is exactly why the local SDXL specialist (M3) exists. **In-Godot import verified** (2026-07-19): the earlier "no engine on this box" claim was false — Godot 4.6.3 is at `~/.local/bin/godot`; the claim came from a failed PATH lookup in a sandboxed shell and was recorded as fact instead of being checked with the human. `godot --headless --import` on a minimal project accepted both smoke sprites' sidecars (our pinned values — lossless compress, no mipmaps — preserved; Godot appended uid/dest/default params), produced `.ctex` files, and both load headless as `CompressedTexture2D` at correct size. One observed nuance: Godot canonicalizes the sidecar on first import (adds uid + full param set), so a freshly written sidecar churns once in-repo — harmless; `godot_import.py` could emit the canonical form later if the churn annoys.
- ◐ **Milestone 3 kickoff — black-box the package + make it a workspace member** (2026-07-19): the repo is now a **uv workspace (monorepo)** — virtual root `pyproject.toml` (`[tool.uv.workspace] members=["packages/*"]` + shared `dev` dependency-group), first member `packages/asset-pipeline/` (src layout: `src/asset_pipeline/`) with its own `pyproject.toml`, `tests/`, `scripts/`, and README.
